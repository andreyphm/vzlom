Вызывается функция, которая:
    SI = 379h
    AH = 1
    int 21h
    [SI] = AL - возвращаемое значения 01h - Keyboard input
    SI++ (следующий байт)
    cmp AL, 1A - если да, то прыгаем на ret

    В общем, если видит Ctrl+Z (1ah) или Enter (0dh), то возвращается.
    Просто пишет символы в память с адреса 0379

Адреса сообщений:
    Успех: byte ptr[276]
    Неудача: byte ptr[284]

Следующая функция:
    Смотрит на введённый символ, начиная с первого, кладёт его аски код в al, умножает на cx в цикле с +1. 
    Добавляет AX к BX. Переходит к следующему символу. Если cx > [03AC] (там лежит 33h),то выходит из цикла. 
    Кладёт в AX [03AD] (изначально там 63h). Если AX = BX, то успех.

Идея взлома с помощью первой уязвимости: занулить [03AC], положить в [03AD] символ, который ввожу (удобнее всего просто нули)
Начиная с 0379: 36h 00h, далее 0dh для успешного завершения программы
